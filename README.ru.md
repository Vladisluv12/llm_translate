# zen-translate

> Перевод сайтов и PDF в режиме разделённого экрана, работающий на локальной AI-модели (Ollama).

Расширение для браузера Firefox/Zen, которое переводит английский и китайский контент на русский с помощью локально запущенной языковой модели. Оригинальная страница остаётся слева, перевод появляется в отдельном окне справа. Облачные API не нужны.

---

## Возможности

- **Разделённый экран** — браузер делится на два окна через `browser.windows` API (нативный split в Zen не имеет публичного API для расширений, поэтому используется изменение размеров окон)
- **Перевод сайтов** — извлекает текстовые блоки из DOM на уровне параграфов, переводит батчами, результаты появляются по мере готовности
- **Перевод PDF** — рендерит PDF через PDF.js, извлекает текст из текстового слоя; для отсканированных страниц использует OCR через Tesseract.js
- **Ленивая загрузка PDF** — переводит только видимые в данный момент страницы; при нехватке RAM автоматически освобождает память canvas-элементов
- **OpenAI-совместимый API** — работает с Ollama из коробки; можно подключить любого провайдера с OpenAI-совместимым API (NVIDIA NIM, Together AI, OpenRouter и др.)
- **Настройка промптов** — полный контроль над системным промптом, батч-промптом и промптом для одиночного параграфа с шаблонными переменными
- **AI Context-Aware режим** — опциональный предварительный проход: модель извлекает тему страницы и ключевые термины, что улучшает единообразие перевода
- **Ограничение запросов и повторные попытки** — токен-бакет rate limiter с экспоненциальным backoff при 429; при ошибке батча автоматически переходит к переводу по одному параграфу
- **Горячая клавиша** — `Alt+A` включает/выключает перевод на активной вкладке (можно изменить в настройках расширений браузера)
- **Определение языка** — автоматически определяет английский или китайский язык источника через franc; Tesseract выбирает соответствующий словарь OCR (`eng` / `chi_sim`)

## Установка

### Требования

- Браузер [Zen](https://zen-browser.app/) или любой браузер на основе Firefox 115+
- [Ollama](https://ollama.com/) запущен локально и загружена хотя бы одна модель:
  ```bash
  ollama pull llama3.1
  # или
  ollama pull mistral:7b
  ```

### Загрузка расширения

```bash
# 1. Клонируйте репозиторий и установите зависимости
git clone https://github.com/Vladisluv12/llm_translate.git
cd llm_translate
npm install

# 2. Соберите расширение
npm run build

# 3. Загрузите в браузер
# Перейдите в about:debugging → Этот Firefox → Загрузить временное дополнение
# Выберите: dist/manifest.json
```

Чтобы собрать zip-архив для распространения:
```bash
npx web-ext build --source-dir dist/ --artifacts-dir web-ext-artifacts/
```

## Использование

1. Откройте любую англоязычную или китайскую страницу
2. Нажмите `Alt+A` или кнопку в панели инструментов → **Translate Page**
3. Браузер делится: оригинал слева, перевод на русский справа
4. Прокрутка любой панели синхронизирует другую

**Для PDF:**
1. Откройте PDF в браузере
2. Нажмите кнопку в панели → **Translate PDF**
3. Откроется новая вкладка: PDF слева, перевод справа
4. Перевод загружается постранично по мере прокрутки

## Настройки

Откройте страницу настроек через попап в панели инструментов или через `about:addons` → Zen Translate → Настройки.

### Провайдер

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| API URL | строка | `http://localhost:11434/v1/chat/completions` | Любой OpenAI-совместимый endpoint |
| API Key | строка | *(пусто)* | Требуется для облачных провайдеров; для Ollama оставьте пустым |
| Модель | строка | `llama3.1` | Название модели, как его видит провайдер |
| Температура | число | `0.1` | Чем меньше — тем детерминированнее перевод |
| Таймаут запроса | секунды | `120` | Увеличьте при медленном локальном инференсе |
| Макс. запросов/сек | число | `5` | Ограничение скорости запросов к API |
| Макс. длина текста | символы | `1800` | Максимальное количество символов в одном запросе |
| Макс. параграфов | число | `10` | Максимальное количество параграфов в одном батче |

### Промпты

Доступные переменные в шаблонах промптов:

| Переменная | Значение |
|---|---|
| `{{from}}` | Название языка источника (например, `English`) |
| `{{to}}` | Название языка перевода (`Russian`) |
| `{{text}}` | Текст одного параграфа |
| `{{json}}` | JSON-массив параграфов для батч-запросов |
| `{{imt_title}}` | Заголовок страницы |
| `{{imt_theme}}` | Тема страницы, извлечённая в AI Context-Aware режиме |

### Дополнительно

| Параметр | По умолчанию | Описание |
|---|---|---|
| AI Context-Aware | выкл | Предварительный проход для извлечения темы и терминологии |
| Язык источника | Авто | Принудительно задать `English` или `Chinese` |

## Архитектура

Расширение следует стандартной структуре Firefox WebExtension с чёткой границей между фоновой логикой и UI-компонентами.

```
manifest.json
src/
  background/
    worker.ts          ← оркестрирует весь pipeline перевода
    openai-client.ts   ← HTTP-клиент для OpenAI-совместимого API
    queue.ts           ← токен-бакет rate limiter + retry с backoff
    text-batcher.ts    ← группирует параграфы в запросы; парсит JSON-ответы
    window-manager.ts  ← разделённый экран через browser.windows
    context-analyzer.ts← AI предпроход для темы и ключевых терминов
  content/
    content.ts         ← инжектируется в каждую страницу; экспортирует __ztExtract, синхронизирует скролл
    extractor.ts       ← чистая функция: DOM → TextBlock[]
  translation/
    translation.html/ts← правая панель; рендерит блоки по мере прихода
  pdf/
    pdf-viewer.html/ts ← рендерер PDF.js + ленивый перевод + синхронизация скролла
    ocr.ts             ← обёртка Tesseract.js с сериализацией вызовов
  shared/
    config.ts          ← схема ProviderConfig; чтение/запись browser.storage
    messages.ts        ← discriminated union для всех сообщений между компонентами
    lang-detect.ts     ← обёртка franc-min; преобразует ISO-коды в 'en' | 'zh'
  popup/  settings/    ← UI панели инструментов и форма настроек
```

**Ключевые паттерны:**

- **Message passing** — всё взаимодействие между content scripts, background worker и окном перевода идёт через типизированный `runtime.sendMessage` / `tabs.sendMessage`. Discriminated union `Message` в `messages.ts` — единственный источник истины для шины сообщений.
- **Токен-бакет + chain-of-responsibility fallback** — `RateLimitedQueue` ограничивает количество запросов в секунду. Если батч-запрос не удался — worker автоматически пытается перевести каждый параграф по отдельности.
- **Observer-driven lazy loading (PDF)** — `IntersectionObserver` с отступом 200 px запускает перевод только когда страница PDF появляется в области просмотра. Сентинел `pagesInProgress` предотвращает дублирование при повторном срабатывании observer.
- **LRU с управлением памятью (PDF)** — `performance.memory` опрашивается каждые 10 с; canvas-элементы страниц за пределами 5 (предупреждение) или 2 (критично) страниц от текущей позиции освобождаются (`width = height = 0`). Tesseract worker завершается при критическом давлении и пересоздаётся по требованию.
- **Серийная очередь OCR** — `OcrManager.recognizePage` выстраивает вызовы в цепочку `Promise`, чтобы однопоточный worker Tesseract никогда не получал параллельные вызовы `recognize()`. При смене языка worker автоматически пересоздаётся.
